# Migration Guide: Metalsmith 2.6.x to 2.7.0

This document chronicles the migration from Metalsmith 2.6.3 to 2.7.0, replacing the `metalsmith-static-files` plugin with the native `statik` method.

## Overview

Metalsmith 2.7.0 introduced the `statik` getter/setter method for pass-through file copying. This native method copies files directly to the destination without plugin processing, eliminating the need for `metalsmith-static-files` for truly static files.

## Key Insight: Static Files vs Bundler Inputs

The migration requires understanding the distinction between:

- **Static files** (images, icons, audio) → Move to `src/assets/` and use `statik`
- **Bundler inputs** (main.css, main.js, styles/) → Keep in `lib/assets/` for processing by `metalsmith-bundled-components`

The `statik` method works with files **relative to `metalsmith.source()`**, so only truly static files should be placed in the source directory.

## Migration Steps

### 1. Restructure Assets

Move only static files to the source directory. Keep bundler entry points in their original location.

```bash
# Move static files TO src/assets/
mv lib/assets/images src/assets/images
mv lib/assets/icons src/assets/icons

# KEEP bundler inputs in lib/assets/
# lib/assets/main.css     (bundler entry point)
# lib/assets/main.js      (bundler entry point)
# lib/assets/styles/      (CSS imports for PostCSS)
```

### 2. Update package.json

Update Metalsmith version and remove the static files plugin:

```json
{
  "dependencies": {
    "metalsmith": "^2.7.0"
  }
}
```

Remove `metalsmith-static-files` from dependencies.

### 3. Update metalsmith.js

Remove the plugin import:

```javascript
// Before
import assets from 'metalsmith-static-files';

// After - remove this line entirely
```

Add the `statik` configuration after `.destination()`:

```javascript
.destination('./build')
.statik(['assets'])  // Static files in src/assets/ copied without processing
```

Remove the assets plugin usage:

```javascript
// DELETE this block:
.use(
  assets({
    source: 'lib/assets/',
    destination: 'assets/'
  })
)
```

### 4. Update Watch Paths

Update watch patterns to include bundler inputs from `lib/assets/`:

```javascript
// Before
.watch(isProduction ? false : ['src/**/*', 'lib/layouts/**/*', 'lib/assets/**/*', 'lib/data/**/*'])

// After
.watch(isProduction ? false : [
  'src/**/*',
  'lib/layouts/**/*',
  'lib/assets/main.css',
  'lib/assets/main.js',
  'lib/assets/styles/**/*',
  'lib/data/**/*'
])
```

### 5. Install Dependencies

```bash
npm install
```

## How statik Works

The `statik` method accepts glob patterns relative to the source directory:

```javascript
// Single pattern
metalsmith.statik('assets');

// Multiple patterns
metalsmith.statik(['assets', 'CNAME', 'robots.txt']);
```

Files matched by these patterns are copied directly to the destination directory, preserving their relative paths. They bypass all plugin processing, making builds faster for binary assets.

## Directory Structure After Migration

```
project/
├── src/
│   ├── assets/          ← Static files (images, icons) - copied via statik()
│   │   └── images/
│   └── (markdown files)
├── lib/
│   ├── assets/          ← Bundler inputs - processed by component bundler
│   │   ├── main.css     (entry point)
│   │   ├── main.js      (entry point)
│   │   └── styles/      (CSS imports)
│   ├── layouts/
│   └── data/
└── build/
    └── assets/          ← Combined output (static + bundled)
```

## What Gets Output to build/assets/

After build, `build/assets/` contains:

1. **Static files** copied via `statik()`:
   - `images/` - from `src/assets/images/`
   - `icons/` - from `src/assets/icons/`

2. **Bundled files** generated by `metalsmith-bundled-components`:
   - `main.css` - bundled from `lib/assets/main.css` + component CSS
   - `main.js` - bundled from `lib/assets/main.js` + component JS

## Lessons Learned

### 1. The `statik` method requires files in the source directory

Unlike `metalsmith-static-files` which could copy from any directory, the native `statik` method only works with files inside the source directory.

### 2. Separate concerns: static files vs bundler inputs

The migration clarified an important architectural distinction:
- **Static files** (`src/assets/`): Images, audio, icons - copied as-is via `statik`
- **Bundler inputs** (`lib/assets/`): CSS/JS entry points and styles - processed by bundler

### 3. Watch paths need explicit bundler input patterns

Since `lib/assets/` is no longer watched entirely, bundler input files need explicit patterns in the watch configuration.
